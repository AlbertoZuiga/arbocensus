# Dependency Management with pip-tools

This backend uses [pip-tools](https://github.com/jazzband/pip-tools) to manage Python dependencies.

## Files

- `requirements.in` — high-level production dependencies (human-maintained)
- `requirements.txt` — compiled, pinned production dependencies (auto-generated)
- `dev-requirements.in` — high-level dev dependencies (human-maintained)
- `dev-requirements.txt` — compiled, pinned dev dependencies (auto-generated)

## Workflow

### Installing dependencies

Install from compiled files (reproducible installs):
```bash
pip install -r requirements.txt          # Production only
pip install -r dev-requirements.txt      # Includes production + dev
```

### Updating dependencies

1. Edit `requirements.in` or `dev-requirements.in` to change versions
2. Recompile:
   ```bash
   pip-compile requirements.in -o requirements.txt
   pip-compile dev-requirements.in -o dev-requirements.txt
   ```
3. Review the `.txt` diff before committing

### Adding a new dependency

1. Add to appropriate `.in` file with minimum version constraint
2. Recompile to resolve the dependency tree
3. Commit both `.in` and `.txt` files

Example:
```bash
# Add to requirements.in:
# django-filter>=23.0

pip-compile requirements.in -o requirements.txt
git add requirements.in requirements.txt
git commit -m "feat(deps): add django-filter for advanced filtering"
```

### Upgrading all dependencies

```bash
pip-compile --upgrade requirements.in -o requirements.txt
pip-compile --upgrade dev-requirements.in -o dev-requirements.txt
```

⚠️ Review the changes carefully — `.txt` files should be reviewed for breaking changes.

## Why pip-tools?

- **Reproducible**: `.txt` files pin exact versions, including transitive deps
- **Maintainable**: `.in` files stay clean and human-readable
- **Transparent**: `pip-compile` shows exact versions resolved + where each dep comes from
- **Safe**: Easy to review what changed before deploying

## Docker

The Dockerfile installs from `dev-requirements.txt` for development, ensuring the same environment locally and in CI.
